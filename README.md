# Трёхпалый самоцентрирующийся захват (3-finger self-centering gripper)

Роботизированный захват с тремя губками, которые сходятся к центру одновременно от **одного** двигателя. За счёт планетарного механизма объект всегда зажимается по центру — это и называется самоцентрированием. Подходит для предметов разной формы: цилиндры, кубики, неровные детали.

Управление **серверное**: команда с веб-панели уходит на сервер (ESP32 в железе / Flask на компьютере для проверки), сервер крутит шаговый двигатель и **хранит состояние** захвата. Панель только отправляет команды и раз в секунду опрашивает статус.

---

## Описание устройства

| Параметр | Значение |
|---|---|
| Тип | трёхпалый самоцентрирующийся захват |
| Двигатель | шаговый NEMA 17 (через драйвер A4988 / DRV8825) |
| Механизм | планетарный: ведущая шестерня → 3 планетарные → 3 плеча кулачков → губки |
| Диапазон захвата | 10–80 мм (настраивается) |
| Усилие захвата | до ~15 Н (зависит от двигателя и материала) |
| Вес | ~150 г |
| Материал деталей | PLA / PETG / ABS, 3D-печать |
| Контроллер | ESP32 |

### Как это работает
1. **Открыто** — кулачки разведены, губки максимально раскрыты.
2. **Захват** — двигатель вращает ведущую шестерню, та крутит три планетарные шестерни, каждая поворачивает своё плечо кулачка → все три губки одновременно идут к центру.
3. **Удержание** — объект зажат строго по центру.

Так как все губки приводятся от одной ведущей шестерни, они двигаются синхронно — отсюда самоцентрирование.

### Детали (все печатаются на 3D-принтере)
- Корпус (держатель мотора) — 1 шт.
- Ведущая шестерня (на валу двигателя) — 1 шт.
- Планетарная шестерня — 3 шт.
- Кулачок — 3 шт.
- Плечо кулачка — 3 шт.
- Губка (со сменной накладкой) — 3 шт.
- Основание — 1 шт.

## Управление

На сервере хранится состояние захвата:
- `percent` — раскрытие в процентах (0 = сжато, 100 = раскрыто);
- `gap_mm` — текущий зазор между губками в мм;
- `moving` — крутится ли двигатель сейчас;
- `status` — `open` / `closed` / `partial` / `moving`.

Веб-панель даёт: кнопки **Сжать / Разжать / Стоп**, **ползунок раскрытия 0–100%** и блок **статуса**, который обновляется раз в секунду.

### Запись состояния в файл

Сервер не только держит состояние в памяти, но и **сохраняет его в файл**, чтобы оно не терялось при перезапуске и оставалась история:

- `state.json` — текущее состояние захвата. Сервер записывает его при каждой команде и читает при запуске. Если сервер перезапустить, захват «помнит» последнее положение.
- `history.log` — журнал всех команд: время, команда и состояние после неё. Пример строки:
  ```
  2026-06-16 20:41:08  set       percent=40 gap=38.0mm status=partial
  ```

На железе (ESP32) то же самое: состояние пишется во внутреннюю flash-память (SPIFFS), файл `/state.json`.

> **Коротко (если спросят на защите):** сервер хранит состояние захвата и записывает его в файлы. `state.json` — текущее положение, сервер читает его при запуске, поэтому состояние не теряется при перезапуске. `history.log` — журнал: каждая команда пишется отдельной строкой со временем, это история работы.

### Команды (эндпоинты сервера)

| Эндпоинт | Что делает |
|---|---|
| `/grip` | сжать губки к центру до упора |
| `/release` | раскрыть губки полностью |
| `/stop` | остановить двигатель в текущем положении |
| `/set?percent=N` | установить раскрытие N% (0–100) |
| `/status` | вернуть текущее состояние (JSON) |

Пример ответа `/status`:
```json
{"percent": 50, "gap_mm": 45, "moving": false, "status": "partial"}
```

## Быстрый старт

### Вариант 1. Проверка на компьютере (без железа)
1. Установить Python и Flask:
   ```
   pip install flask
   ```
2. Запустить сервер:
   ```
   python server.py
   ```
3. Открыть в браузере: `http://127.0.0.1:5000`

### Вариант 2. На железе (ESP32)
1. Открыть `esp32_grip.ino` в Arduino IDE.
2. Установить плату «ESP32 Dev Module» и библиотеку **AccelStepper** (через Менеджер библиотек).
3. Прошить ESP32.
4. На телефоне/ноутбуке подключиться к Wi-Fi сети:
   - Сеть: **Grip3Control**
   - Пароль: **12345678**
5. Открыть в браузере: `http://192.168.4.1`

> Число шагов от полного сжатия до раскрытия задаётся в прошивке (`STEPS_FULL`) — подбирается под передаточное число планетарного механизма и микрошаг драйвера.

## Подключение электроники (ESP32 + драйвер шагового)

| ESP32 | Драйвер (A4988/DRV8825) |
|---|---|
| GPIO26 | STEP |
| GPIO27 | DIR |
| GPIO25 | EN (enable) |
| GND | GND (логики) |

Питание двигателя NEMA 17 — отдельное (обычно 12 В) на силовую часть драйвера. Не забудь общий GND между ESP32 и драйвером.
